Description

Photon mapping is a two-pass global illumination algorithm to simulate realistic lighting. Compared to ray tracing, can effectively handle complex light interactions such as caustics and diffuse interreflections. Also, it's less computationally expensive than traditional ray tracing in general, because it's precomputes the global illumination.

We implemented photon mapping from scratch for our final project.

Our photon mapping implementation includes the following:

  1. Photon Maps: We emit photons from area lights and calculate each photon's throughput upon hitting a surface. The photon's position, direction, and throughput are stored in a KD Tree for efficient retrieval. We maintain two photon maps: one for global illumination and one for caustics. The global illumination photon map stores all the photons that hit a diffuse surface, and the caustics photon map stores all the photons that hit a diffuse surface and then a specular surface in sequence.
  2. Gathering: At each intersection point (where light hits a surface), we search for all photons within a radius of 0.5 units around the intersection point.
  3. Rendering: (BRDF, reflectance equations, etc) to do rendering and output final image.

We use the following techniques to improve sampling quality and reduce computational time:

  1. KD Tree: We use KD Tree to store the photons, so that when finding nearest K photons around an intersection point, we can use tree search to reduce searching time.
  2. Cosine Importance Sampling: We implement cosine importance sampling to prioritize sample directions that contribute more to the desired illumination, improving the accuracy and efficiency of our sampling process.
  3. Russian Roulette: We use RR to probabilistically terminate paths when their throughput becomes very small, which adaptively focuses the computational resources on more important paths.
